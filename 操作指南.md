# DriftLog 启动与开发说明

本文档说明如何在本地安装依赖并启动 DriftLog（Electron + electron-vite）。**不修改项目代码**即可完成日常开发启动。

---

## 环境要求

- **macOS**（本项目为菜单栏应用，面向 Mac）
- **Node.js**：建议 LTS 版本（与 `package.json` 中工具链兼容即可）
- **npm**：随 Node 安装即可

---

## 一、安装依赖

在项目根目录（`DriftLog/`）执行：

```bash
cd /path/to/DriftLog
npm install
```

说明：

- 首次克隆或删除 `node_modules` 后需要执行。
- 若包含原生模块（如 `better-sqlite3`），安装脚本会配合 `electron-builder` 做必要处理；若安装失败，可查看终端报错后按提示重试（例如网络或本机编译环境）。

---

## 二、在 Cursor 内置终端中启动（重要）

Cursor 基于 Electron，其**集成终端**里常会设置环境变量：

`ELECTRON_RUN_AS_NODE=1`

该变量会让通过 `electron` 启动的进程以「纯 Node」方式运行，**Electron 主进程 API（如 `app`）不可用**，表现为启动失败或行为异常。

**推荐做法：启动前先取消该变量，再运行开发命令：**

```bash
unset ELECTRON_RUN_AS_NODE
npm run dev
```

也可一行执行：

```bash
unset ELECTRON_RUN_AS_NODE && npm run dev
```

在 **系统自带终端（Terminal.app）或 iTerm** 中，若未设置上述变量，通常可直接：

```bash
npm run dev
```

---

## 三、开发模式在做什么

执行 `npm run dev` 后（脚本为 `electron-vite dev`），一般会看到：

1. **主进程 / preload** 被编译到 `out/` 目录。
2. **渲染进程** 由 Vite 提供开发服务，终端会打印类似：
   - `Local: http://localhost:5173/`（若 5173 被占用，会自动换端口，如 `5174`）。
3. **Electron 应用窗口**：本应用为菜单栏形态，主窗口默认可能不常驻显示；**通过菜单栏右侧托盘图标** 点击打开面板（具体以当前产品行为为准）。

若需在浏览器中单独查看前端页面，可打开终端里打印的 `http://localhost:xxxx/`（仅用于调试 UI，**完整能力仍以 Electron 内为准**）。

## 四、常见问题

### 1. 端口被占用（5173 / 5174 等）

终端可能出现 `Port 5173 is in use, trying another one...`，属正常。若怀疑旧进程未退出：

- 结束当前终端里的 `npm run dev`（`Ctrl+C`）。
- 若仍异常，可在系统「活动监视器」中结束与 `electron-vite` 或本项目 `Electron` 相关的进程，或关闭多余终端后重试。

### 2. 托盘图标不明显或看不到

- 菜单栏右侧图标较多时，新图标可能被挤到「刘海」或折叠区域附近，可尝试整理其他菜单栏图标腾出空间。
- 托盘资源应位于项目 `resources/`（如 `iconTemplate.png`）；若缺失，应用可能有备用逻辑，但建议保留资源文件以便正常显示。

### 3. `npm warn Unknown env config "devdir"`

来自本机 npm 配置（`.npmrc` 等），一般**不影响**本项目启动；若需消除警告，可自行检查 npm 配置中是否含有无效项。

## 五、其他常用命令（参考）

| 命令 | 说明 |
|------|------|
| `npm run dev` | 开发模式（热更新等由 electron-vite 提供） |
| `npm run build` | 构建主进程 / preload / 渲染产物 |
| `npm run preview` | 预览构建结果（以项目配置为准） |

打包分发（如 DMG）见 `package.json` 中 `pack` / `dist` 等脚本，需时再使用。

---

## 六、小结（最短启动步骤）

在 **Cursor 终端** 中：

```bash
cd /path/to/DriftLog
npm install          # 仅首次或依赖变更时
unset ELECTRON_RUN_AS_NODE && npm run dev
```

启动成功后，在 **macOS 菜单栏右侧** 查找 DriftLog 托盘图标并点击使用。
